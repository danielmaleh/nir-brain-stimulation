## Software overview

The experiment was orchestrated by a purpose-built, browser-based control
system comprising three coordinated components: (i) a behavioural
reaction-time task, (ii) firmware controlling the transcranial
photobiomodulation (tPBM) device, and (iii) a marker bridge that time-stamped
all experimental events into the electroencephalography (EEG) recording. The
client applications were implemented in vanilla HTML/CSS/JavaScript and executed
in a Chromium-based browser; the device firmware was written in C++ for an
Arduino microcontroller; and the marker bridge was implemented in Python. The
complete source code is openly available (see *Software availability*).

## Behavioural reaction-time task

The behavioural task was a simple auditory reaction-time (finger-tapping)
paradigm implemented as a single-page web application. On each trial an
auditory stimulus was presented and participants were instructed to respond as
quickly as possible by pressing the space bar. The stimulus was a 600 Hz sine
tone (~180 ms duration) generated programmatically through the Web Audio API
with a smooth amplitude envelope (10 ms attack) to avoid onset transients
(clicks) that could otherwise act as an additional, sharper cue.

The inter-stimulus interval was 5.0 s plus a uniformly distributed random jitter
of 0–2.0 s, measured from the participant's previous response (or, on omission
trials, from the close of the previous response window), so that stimulus onsets
were temporally unpredictable. After each tone a fixed 2.0 s response window was
opened: a space-bar press within this window was recorded as a valid response
and its reaction time logged; a press occurring before the tone was recorded as
an anticipatory false alarm and reset the stimulus schedule; and the absence of
any press within the window was recorded as an omission (no-response) trial,
after which the run continued automatically.

Response timing was captured with high precision. Key events were intercepted in
the browser's event capture phase and time-stamped with the
`performance.now()` monotonic clock (sub-millisecond resolution, immune to
wall-clock adjustments), and keyboard auto-repeat events were filtered out.
Reaction times were computed relative to the *audible* onset of the tone rather
than to the scheduling instant: the onset time was derived from the Web Audio
hardware clock (including the envelope attack) and mapped into the same
`performance.now()` timeline as the keypresses, so that reaction times reflected
the true stimulus-to-response latency.

## Session structure and randomisation

Each session consisted of four runs presented in a per-participant randomised
order, covering all four conditions exactly once: heating control, 10 Hz NIR,
40 Hz NIR, and a silent wrist-electromyography (EMG) run. The three
stimulation/task runs each lasted 40 s; the wrist-EMG run lasted 20 s and
contained no auditory stimuli or responses. Consecutive runs were separated by a
configurable rest interval (default 180 s). The randomised condition sequence
was generated at session start and logged. During active runs the participant-
facing display presented only a minimal fixation element, and participants were
instructed to keep their eyes closed, in order to preserve blinding.

## Photobiomodulation device firmware and control

The tPBM device was driven by firmware running on an Arduino microcontroller.
Pulsed near-infrared (850 nm) stimulation at 10 Hz and 40 Hz (50 % duty cycle)
was generated by a hardware timer (Timer1) operating in clear-timer-on-compare
mode, independently of the main program loop and of serial communication. With a
prescaler of 64 (4 µs per tick) the compare values produced exact half-periods
of 50.000 ms and 12.500 ms, yielding pulse frequencies of 10.00 Hz and 40.00 Hz
derived directly from the 16 MHz crystal oscillator and therefore accurate to
well within ±0.5 Hz. Generating the pulse train in the timer interrupt ensured
that the inter-pulse timing was unaffected by jitter from temperature polling or
serial output. Every pulse edge was time-stamped in the interrupt and streamed
over the serial link for logging.

Skin-contact temperature was monitored continuously with a digital 1-Wire
temperature sensor (DS18B20) polled at 5 Hz. A latching safety cut-off
immediately de-energised all actuators if the measured temperature reached
40.0 °C, and the firmware rejected the sensor's power-on artefact value to
prevent spurious trips; a missing or disconnected sensor also forced the device
into the safe (stimulation-disabled) state. In the heating-control condition the
warming element was regulated by an on–off thermostat about a 37.5 °C set point
with 0.5 °C hysteresis. To equate the thermal
percept between conditions, the heating-control set point was not fixed but was
matched to the temperature the NIR stimulation itself produced: in a separate
calibration procedure the 10 Hz and 40 Hz conditions were each run for the
experimental run duration on a fixed contact set-up while the contact
temperature was recorded, and the temperature *rise* the NIR
produced (the plateau minus the pre-stimulation baseline, averaged across the
two frequencies, which were near-identical at 50 % duty cycle) was stored. During
the heating-control condition the warming element was regulated by a closed-loop
proportional–integral–derivative (PID) controller (updated at 5 Hz from the
temperature sensor, its output applied as a time-proportioned duty cycle) to
reproduce that same rise **above each participant's own baseline skin
temperature**, measured at the start of the run — matching the *change* in
contact temperature the NIR causes rather than an absolute set point, since skin
temperature varies between participants and is far above the calibration
ambient. Consistent with the blinding
design, the invisible 850 nm emission carried no accompanying visible indicator,
and all condition-indicator lights were extinguished whenever stimulation was
active. The firmware
was driven over a USB serial interface and emitted a time-stamped log of all
state transitions, pulses, temperature readings, heater cycles, and safety
events.

## Integration of task and stimulation

The behavioural task application communicated with the device directly from the
browser using the Web Serial API. At the start of each run the application
selected the corresponding stimulation condition on the device and initiated
stimulation; at the end of each run (or on manual abort) stimulation was
stopped. Stimulation therefore followed the task's timing (40 s runs), and no
near-infrared or heating output was delivered during the wrist-EMG run. This
integration ensured that the delivered stimulation and the behavioural task were
automatically co-registered within each run without manual intervention.

## Event synchronisation with the EEG recording

Because web browsers cannot natively use the Lab Streaming Layer (LSL), event
markers were relayed through a lightweight local bridge. Both the behavioural
task and the device-control layer sent event notifications over a localhost
WebSocket to a Python process (using `pylsl`), which published them as an LSL
marker stream. This stream was received by the EEG acquisition software (NIC2)
and written into the recording, so that all events shared the EEG timebase. The
localhost relay added an approximately constant delay of ~1–3 ms, and LSL
synchronised the streams to the order of 1 ms, which is well within the
tolerance required for the present analyses.

Markers were recorded as integer event codes. The marked events comprised, from
the behavioural task: run onset and offset (with condition), tone onset, valid
responses, anticipatory false alarms, and omissions; and from the device:
stimulation onset and offset (with condition), heater switching, and safety
events. A complete code–event lookup table is provided with the software.

## Data acquisition and storage

Behavioural data — including per-trial reaction times, false alarms, omissions,
and event timestamps — were logged in real time and continuously persisted to
browser local storage to guard against data loss, and were exportable as
comma-separated-value files. Continuous EEG (and wrist EMG) were acquired with a
32-channel Enobio system and recorded by NIC2 with the embedded event markers as
described above. Behavioural, electrophysiological, and marker streams were
aligned offline through the shared LSL timebase and the event-code table.

## Software availability

The full experiment-control software — the behavioural task application, the
device firmware, the marker bridge, and the analysis code/lookup tables — is
openly available at <https://github.com/danielmaleh/nir-brain-stimulation>.
